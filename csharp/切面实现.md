# 实现一：KingAOP

https://github.com/AntyaDev/KingAOP/

实现思路：Dynamic

在 C# 4.0 之前，C# 是静态类型的。变量的类型在编译时确定，如果调用不存在的方法，编译器会直接报错。

引入 `dynamic` 后，

```
dynamic obj = GetSomeObject();
obj.DoSomething(); // 编译通过，运行时检查
```

- **编译时**：编译器不检查 `obj` 是否有 `DoSomething` 方法，它只生成一段“稍后在运行时解析”的代码。
- **运行时**：CLR（公共语言运行时）之上的 **DLR（动态语言运行时）** 介入。DLR 会查看 `obj` 的真实类型，查找 `DoSomething` 方法，并调用它。

**核心问题：** DLR 如何知道某个对象支持哪些动态操作？如何调用？ **答案：** 通过 `IDynamicMetaObjectProvider` 接口。

### IDynamicMetaObjectProvider 接口

这是 DLR 的“入口协议”。如果一个类型想要完全控制其动态行为（而不仅仅是依赖默认的反射），它必须实现这个接口。

```
public interface IDynamicMetaObjectProvider
{
    DynamicMetaObject GetMetaObject(Expression parameter);
}
```

### GetMetaObject 方法详解

这是接口中唯一的方法，也是动态绑定的**核心工厂**。

- **输入 (`Expression parameter`)**: 代表当前对象实例的表达式树。
- **输出 (`DynamicMetaObject`)**: 这是一个描述“如何对该对象执行操作”的对象。

**它的作用不是直接执行操作，而是返回一组“指令”（表达式树），告诉 DLR 当发生特定操作（如获取属性、调用方法）时该怎么做。**

当你执行 `dynamicObj.SomeProperty` 时：

1. **Call Site (调用点)**: 编译器生成的代码在运行时首次遇到这个操作。
2. **Binder (绑定器)**: DLR 的绑定器接管控制权。
3. 检查接口: 绑定器检查 dynamicObj的实际类型是否实现了 IDynamicMetaObjectProvider
   - **如果是**: 调用 `GetMetaObject`。
   - **如果否**: 使用标准的反射机制查找成员。
4. **获取 MetaObject**: `GetMetaObject` 返回一个 `DynamicMetaObject`。
5. 绑定逻辑：DynamicMetaObject内部包含了一个 BindingRestrictions（绑定限制）和 Expression（执行逻辑）。
   - 例如：它可能说“如果对象类型是 X，且包含属性 Y，则执行获取 Y 值的 IL 代码”。
6. **缓存**: DLR 会将这个绑定结果缓存起来。下次再对同类型的对象执行相同操作时，直接使用缓存的指令，无需再次调用 `GetMetaObject`，从而提高性能。
7. **执行**: 编译并执行生成的指令。



如果你需要极致的性能控制，或者不想继承 `DynamicObject`，可以直接实现接口。这通常用于实现特定的 DSL（领域特定语言）。

```
public class CustomDynamic : IDynamicMetaObjectProvider
{
    public DynamicMetaObject GetMetaObject(Expression parameter)
    {
        // 这里需要手动构建表达式树，非常复杂
        // 通常返回一个自定义的 DynamicMetaObject 子类
        return new CustomMetaObject(parameter, this);
    }
}

public class CustomMetaObject : DynamicMetaObject
{
    public CustomMetaObject(Expression expression, CustomDynamic value) 
        : base(expression, BindingRestrictions.Empty, value) { }

    public override DynamicMetaObject BindGetMember(GetMemberBinder binder)
    {
        // 手动定义获取成员时的逻辑表达式
        // 例如：直接返回一个常量，或者访问特定字段
        var restrictions = BindingRestrictions.GetTypeRestriction(Expression, LimitType);
        var getValue = Expression.Property(Expression, "SomeHardcodedProperty"); 
        
        return new DynamicMetaObject(getValue, restrictions);
    }
}
```

你可能会问：*“为什么不让 dynamic 直接全部用反射？为什么要搞出 MetaObject 这么复杂的东西？”*

1. 性能 (Performance)
   - 纯反射很慢。
   - `GetMetaObject` 返回的是**表达式树**，DLR 会将其编译成 IL 代码并**缓存**。
   - 第二次调用时，直接执行缓存的 IL，速度接近静态调用。
2. 灵活性 (Flexibility)
   - 反射只能操作实际存在的成员。
   - 通过 `IDynamicMetaObjectProvider`，对象可以**虚构**成员。例如 `ExpandoObject` 内部只是一个字典，但它可以通过 MetaObject 让外界觉得它拥有真实的属性。
3. 语言互操作性 (Interop)
   - DLR 的设计初衷是为了让 .NET 能与其他动态语言（IronPython, IronRuby）无缝交互。
   - Python 的对象模型和 C# 不同。通过 `GetMetaObject`，Python 对象可以告诉 C# 的 DLR：“当你调用我的 `+` 运算符时，请执行这段 Python 逻辑”。

## 实现方式

1.定义切面

2.把切面写到业务类

3.业务类集成IDynamicMetaObjectProvider接口，并实现，返回包含GetMetaObject方法，返回包含切面逻辑的DynamicMetaObject子类CustomMetaObject

4.CustomMetaObject类实现BindInvokeMember方法，构造包含切面逻辑的表达式树。

```C#
///切面定义
class HelloWorldAspect : OnMethodBoundaryAspect
{
    public override void OnEntry(MethodExecutionArgs args)
    {
        Console.WriteLine("OnEntry: Hello KingAOP");
    }

    public override void OnExit(MethodExecutionArgs args)
    {
        Console.WriteLine("OnExit: Hello KingAOP");
    }
}

//业务类
class HelloWorld : IDynamicMetaObjectProvider
{
    [HelloWorldAspect]  //切面以“特性”的形式使用，类似于Java注解
    public void HelloWorldCall()
    {
        Console.WriteLine("Hello World");
    }

    public DynamicMetaObject GetMetaObject(Expression parameter)
    {
        return new AspectWeaver(parameter, this);
    }
}

//实际调用（需要将对象声明为dynamic，否则只是普通对象）
dynamic helloWorld = new HelloWorld();
helloWorld.HelloWorldCall();
```

可以看出其实现逻辑是通过把业务类变成动态类，实现额外操作的注入。通过GetMetaObject返回动态类执行逻辑对象AspectWeaver（额外的操作将通过AspectWeaver注入）

接下来看AspectWeaver类(只展示核心逻辑):

```c#
public class AspectWeaver : DynamicMetaObject{
    //....
    
     public AspectWeaver(Expression expression, object obj)//绑定对象
            : base(expression, BindingRestrictions.Empty, obj)
    {
        _objType = obj.GetType();
    }
    
    public override DynamicMetaObject BindInvokeMember(InvokeMemberBinder binder, DynamicMetaObject[] args)
        {
            var metaObj = base.BindInvokeMember(binder, args);

            var argsTypes = GetMethodArgsTypes(metaObj);
            var method = _objType.GetMethod(binder.Name,//反射，获取类方法
                BindingFlags.DeclaredOnly | BindingFlags.Instance | BindingFlags.Public | BindingFlags.NonPublic,
                null,
                argsTypes,
                null);

            if (method != null && method.IsDefined(typeof(IMethodAspect), false))//判定切面（也就是特性是否存在）
            {
                var aspects = RetrieveAspects(method);//检索是否存在切面

                if (method.IsDefined(typeof(OnMethodBoundaryAspect), false))//是否存在方法类切面
                {
                    metaObj = new BoundaryAspectGenerator(Value, metaObj, aspects.Cast<OnMethodBoundaryAspect>(), method, args, argsTypes).Generate();//切面相关的额外操作表达式全都写在这个类中
                }
                else
                {
                    metaObj = new InterceptionAspectGenerator(Value, metaObj.Restrictions, aspects.Cast<MethodInterceptionAspect>(), method, args, argsTypes).Generate();
                }
            }
            return metaObj;
        }
    
    //....
}
```

切面逻辑注入类BoundaryAspectGenerator

```C#
internal class BoundaryAspectGenerator
{
    //...
    public BoundaryAspectGenerator(object instance, DynamicMetaObject metaObj, IEnumerable<OnMethodBoundaryAspect> aspects,
            MethodInfo method, IEnumerable<DynamicMetaObject> args, IEnumerable<Type> argsTypes)//对应切面OnMethodBoundaryAspect
        {
            _rule = metaObj.Restrictions;
            
            var methExecArgs = new MethodExecutionArgs(instance, method, new Arguments(args.Select(x => x.Value).ToArray()));

            _aspectCalls = new AspectCalls(metaObj.Expression, aspects, args, methExecArgs, argsTypes.Any(t => t.IsByRef));
        }
    
    public DynamicMetaObject Generate()  //构建切面执行表达式，并返回DynamicMetaObject对象
        {
            Expression method = Expression.Block(
            new[]
            {
                AssignMethodArgsByRetValue(_aspectCalls.ArgsExpression, _aspectCalls.RetMethodValue),
                _aspectCalls.EntryCalls.First(),
                Expression.TryCatchFinally(
                _aspectCalls.OriginalCall,
                _aspectCalls.ExitCalls.First(),
                GenerateCatchBlock(_aspectCalls.ArgsExpression, _aspectCalls.ExceptionCalls.First(), _aspectCalls.RetMethodValue))
            });

            for (int i = 1; i < _aspectCalls.EntryCalls.Count; i++)
            {
                method = Expression.Block(
                new[]
                {
                    _aspectCalls.EntryCalls[i],
                    Expression.TryCatchFinally(Expression.Block(method, _aspectCalls.SuccessCalls[i]), _aspectCalls.ExitCalls[i])
                });
            }

            return new DynamicMetaObject(Expression.Block(new[] { _aspectCalls.RetMethodValue }, method, _aspectCalls.RetMethodValue), _rule);
        }
}
```

# 实现二：aspect-injector

https://github.com/pamidur/aspect-injector

**编译时中间语言（IL）编织**

也是编辑IL代码，不过作者做了封装，用户只需使用即可

例子：（方法执行计时）

```
using AspectInjector;
using System;
using System.Diagnostics;
using System.Threading.Tasks;
using System.Reflection;

[Aspect(Scope.Global)]
public class PerformanceMonitorAttribute : Attribute
{
    [Around]
    public async Task<object> HandleAsync(AspectEventArgs args)
    {
        var stopwatch = Stopwatch.StartNew();
        try
        {
            // 关键：使用 ProceedAsync 等待异步方法真正完成
            return await args.ProceedAsync();
        }
        catch (Exception ex)
        {
            // 异常也记录耗时
            LogPerformance(args, stopwatch, ex);
            throw;
        }
        finally
        {
            // 如果没有异常，在 finally 中记录
            if (stopwatch.IsRunning)
            {
                LogPerformance(args, stopwatch, null);
            }
        }
    }

    // 同步方法兜底（可选）
    [Around]
    public object Handle(AspectEventArgs args)
    {
        var stopwatch = Stopwatch.StartNew();
        try
        {
            return args.Proceed();
        }
        catch (Exception ex)
        {
            LogPerformance(args, stopwatch, ex);
            throw;
        }
        finally
        {
            if (stopwatch.IsRunning)
            {
                LogPerformance(args, stopwatch, null);
            }
        }
    }

    private void LogPerformance(AspectEventArgs args, Stopwatch sw, Exception ex)
    {
        sw.Stop();

        // 获取完整方法标识：命名空间。类名。方法名
        string methodFullName = GetMethodFullName(args);

        // 构建日志信息
        var logMessage = new
        {
            Timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss.fff"),
            Class = args.Type?.FullName ?? "Unknown",
            Method = args.Name,
            FullMethod = methodFullName,
            ElapsedMs = sw.ElapsedMilliseconds,
            ElapsedTicks = sw.ElapsedTicks,
            IsAsync = args.ReturnType?.IsAssignableFrom(typeof(Task)) == true,
            Exception = ex?.Message
        };

        // 输出日志（可替换为 NLog/Serilog）
        WriteLog(logMessage);
    }

    private string GetMethodFullName(AspectEventArgs args)
    {
        // 格式：Namespace.ClassName.MethodName
        var typeName = args.Type?.FullName ?? "UnknownType";
        var methodName = args.Name ?? "UnknownMethod";
        return $"{typeName}.{methodName}";
    }

    private void WriteLog(object logData)
    {
        // 简单控制台输出（生产环境请替换为日志框架）
        Console.WriteLine($"[PERF] {logData}");
        
        // 或者使用结构化日志（推荐）
        // _logger.LogInformation("性能监控：{@PerfData}", logData);
    }
}
```

```
public class UserService
{
    [PerformanceMonitor]
    public async Task<User> GetUserByIdAsync(int id)
    {
        await Task.Delay(100);
        return new User { Id = id, Name = "张三" };
    }

    [PerformanceMonitor]
    public async Task<List<User>> GetAllUsersAsync()
    {
        await Task.Delay(200);
        return new List<User>();
    }

    [PerformanceMonitor]
    public void SyncMethod()
    {
        Thread.Sleep(50);
    }
}

    }
}
```
